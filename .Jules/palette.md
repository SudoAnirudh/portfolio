## 2024-06-16 - Contact Form Semantic HTML & ARIA Link Labels
**Learning:** React/Next.js forms sometimes omit explicit `<label>` elements in favor of just placeholders. This breaks accessibility for screen reader users as they navigate forms. Additionally, when using text-based icon fonts like material-symbols, the icon text itself (e.g., "code") can be read out by screen readers on icon-only links, which is confusing if the link is a social icon.
**Action:** Always add `<label className="sr-only">` to inputs that don't have visual labels, and add `aria-invalid` / `aria-describedby` attributes to associate error messages with inputs. For icon-only links containing text-based icon fonts, apply `aria-hidden="true"` to the inner icon span and `aria-label="..."` to the parent `<a>` tag.

## 2025-06-17 - [Hoist Static Maps and Derived Arrays out of React Components]
**Learning:** In components consuming static array data (like `src/components/Skills.tsx`), creating large derived arrays (e.g., using `flatMap` and `split`) and instantiating large mapping objects (e.g., `skillDetails` and `iconMap`) inside the component function body introduces significant, redundant memory allocation and overhead on every render cycle. Micro-benchmarking confirmed that hoisting these static structures to the module level reduces iteration overhead by ~99%.
**Action:** When working with static configuration or data that doesn't depend on component state or props, always extract it to module-level constants. This avoids unnecessary object creation and processing during re-renders, resulting in smoother UI interactions.


## 2024-11-21 - [Retro UI Accessibility]
**Learning:** Custom UI components mimicking retro operating systems (like winamp clones) often rely entirely on visual cues like "_" or "X" for window controls, and icon-only buttons for playback, leading to complete inaccessibility for screen reader users. Additionally, using text-based icon fonts (e.g. material-symbols-outlined) inside these buttons causes screen readers to read the icon's literal string name (e.g., "play_arrow") instead of a semantic label. Custom form inputs like retro volume sliders also frequently lack programmatic association.
**Action:** When creating or auditing custom retro interfaces, explicitly add `aria-label` to all icon-only or symbolic interactive controls (minimize, close, playback). For buttons containing text-based icon fonts, always pair the outer `aria-label` with `aria-hidden="true"` on the inner icon element to prevent the icon's literal text from being announced. Ensure all custom form elements are explicitly associated with their labels using `htmlFor` and `id` to maintain semantic structure within the styled presentation.
